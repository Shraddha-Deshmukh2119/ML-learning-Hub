Hierarchical Clustering

Building nested clusters by merging or splitting them

What is Hierarchical Clustering?

Hierarchical clustering is an unsupervised learning algorithm that creates a hierarchy of clusters. It builds nested clusters by progressively merging or splitting them. This approach provides more information about the data structure than flat clustering methods like K-means.

There are two main types of hierarchical clustering:

  • Agglomerative (bottom-up): Starts with each data point as a separate cluster and progressively merges the closest clusters until only one cluster remains
  • Divisive (top-down): Starts with all data in one cluster and progressively splits them until each data point forms its own cluster

The results of hierarchical clustering are typically visualized as a dendrogram (tree diagram), which shows the sequence of cluster merges or splits and the distance at which each fusion or split occurs.

Linkage Criteria:

To determine which clusters to merge or split, various linkage criteria are used:

  • Single linkage: Distance between the closest members of different clusters
  • Complete linkage: Distance between the furthest members of different clusters
  • Average linkage: Average distance between all pairs of members from different clusters
  • Ward's method: Minimizes the increase in variance for the clusters being merged

Advantages:

  • No need to specify the number of clusters beforehand
  • Produces an informative hierarchy of clusters
  • Can produce small clusters, which may be useful when studying rare events
  • Works well with various distance metrics

Applications:

  • Taxonomy creation in biology
  • Document clustering
  • Customer segmentation
  • Social network analysis
  • Genetic sequence analysis
